# GOAL
The goal is to generate a coding guidelines for programming which will result in cleaner, performant code, and coding expirience.

# Requierments
I work on performance critical software, were having secure, performant code is paramount.
Writing code that is "fast" is really important.
What code is "fast":
 - Code that reduces the amount of actual cpu instructions
 - Code that is CPU friendly, low branching when possible, high certainty of branch predection to avoid wasting cpu cylcle, etc

I value using the right data type for each ocassion the most performant data type the current project demands.
High performance algorithms.

My personal programming style is a combination of OOP with imperative/procedural programming and functional programming.
What I mean by that is that when possible I try to make everything I can a pure functional method. That way I get deterministic
output for the same input. Sometimes that ends in fully functional programs, and other times I create OOP programs that contain
as much pure functionl methods as possible so as to have a high deree of certainty that everyrthing works as intended, deterministic
output for a given input.

I like to use cutting edge patterns to create the best code possible.
